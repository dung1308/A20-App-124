## Pain-Point Driven Feature Skills

When proposing or implementing new features, optimize for:
- Trust: make AI reasoning visible.
- Recovery: let users retry, revise, or escalate safely.
- Clarity: show what happened and why.
- Speed: prefer features that can be shipped in under 3 days.
- Safety: never expand beyond approved admissions boundaries.

For each proposed feature:
1. Name the pain point.
2. Describe the smallest useful fix.
3. List backend files likely affected.
4. List frontend files likely affected.
5. Define fallback behavior.
6. Define test/verification steps.
